# ShoDumo Public Site — промпт для Claude Code

> Запусти Claude Code в порожній папці `shodumo-web` і встав це.

---

Побудуй публічний сайт ShoDumo — платформи-афіші міні-івентів у Львові. Це Фаза 1 фронт: він споживає готовий REST API (NestJS, base URL з `.env`). Дизайн-референс уже є (mobile-first + desktop): головна-стрічка та сторінка події. Стек і SEO-вимоги нижче — суворо дотримуйся.

## Стек (за техспекою)
- **HTML + SCSS + vanilla JS**, збірка через **Gulp** (мініфікація, бандлінг, autoprefixer, оптимізація зображень, sourcemaps, live-reload через BrowserSync)
- **БЕЗ важких фреймворків** (без React/Vue) — легка статика заради швидкості й SEO
- Fetch до API через тонкий клієнт-обгортку (`src/js/api.js`)
- Конфіг API base URL через `.env` → інжектиться в білд (gulp + dotenv)

## SEO — критично (головний канал залучення = органіка Google)
- Сторінки подій **рендеряться з сервера / пре-рендеряться** (SSR або build-time pre-render зі списку published-подій через `GET /events`), щоб контент був у HTML, а не тільки на клієнті
- Статичні сторінки (головна-лендінг, про нас) — звичайна Gulp-збірка
- На кожній сторінці події: `<title>`, meta description, **Open Graph** теги, **JSON-LD `Event`** schema.org (назва, дата, локація, ціна, організатор), canonical URL
- ЧПУ-маршрути: `/lviv`, `/lviv/events?...`, `/event/:slug`, `/organizer/:id`
- Згенерувати `sitemap.xml` (з опублікованих подій) і `robots.txt`


## Екрани (Фаза 1)
1. **Головна — стрічка подій:** хедер (лого, пошук, місто, Вхід, Створити подію), фільтр-чіпи, бічні фільтри (десктоп), сітка карток (cover, дата, бейдж ціни, назва, локація, "N ідуть"), перемикач Список/Карта, пагінація. Mobile: таб-бар знизу.
2. **Сторінка події:** cover, бейджі (ціна+категорія), заголовок, дата/час, адреса, міні-карта, опис, картка організатора, "схожі події". Десктоп — права липка панель з кнопками "Піду"/"Зберегти".
3. **Карта подій:** Leaflet, піни, прев'ю події, геолокація "поруч зі мною".
4. **Профіль організатора:** avatar, bio, посилання, стрічка його подій.
5. **Модалка Вхід/Реєстрація:** таби, email+пароль; зберігання access/refresh токенів; авто-refresh при 401.

## Інтеграція з API
- `GET /events` (фільтри city, date, category, isPaid, near=lat,lng + пагінація) → стрічка
- `GET /events/:slug` → сторінка події (+ pre-render)
- `GET /organizers/:id`, `GET /cities`, `GET /categories`
- `POST /auth/register|login|refresh` → модалка, токени в localStorage
- `POST /events/:id/attend`, `DELETE /events/:id/attend` → "Піду"/"Зберегти" (вимагає логіну → відкрити модалку)
- Дії, що потребують акаунта, перехоплюють і піднімають модалку логіну, після успіху продовжують дію

## Якість / UX
- Адаптив: mobile-first, брейкпоінти під desktop 1440
- Стани: skeleton-завантаження, порожня стрічка, помилка, офлайн
- Доступність: семантичний HTML, alt, фокус-стани, aria для модалки
- Lazy-load зображень, оптимізовані cover'и
- Тостери підтверджень ("Додано в Піду")
- README: dev (`gulp` watch), prod-білд, pre-render, налаштування `.env`

Почни з: gulp-пайплайн + структура + api.js → головна-стрічка з реальними даними → сторінка події з SSR/pre-render + SEO-теги → карта → модалка auth → профіль організатора. Після кожного блоку — білд, щоб ловити помилки рано.
