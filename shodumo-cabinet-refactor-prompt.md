# ShoDumo — Рефактор + UX кабінету (промпт для Claude Code)

> Запусти Claude Code в корені monorepo `shodumo`. Основне — **`apps/cabinet`**; додатково створи `CLAUDE.md` (репо-рівень).
> Рефактор без зміни бізнес-логіки. Material і нові UX-фічі мають вписатись у наявний дизайн (кольори/токени), а не приносити дефолтний вигляд Material.

---

## 0. CLAUDE.md (конвенції — щоб не повторювати)
Створи `CLAUDE.md` у корені репо з розділами для `apps/web` і `apps/cabinet`. Зафіксуй прийняті домовленості:
- **Структура компонентів (Angular):** кожен компонент — окремі файли `*.component.ts` / `*.component.html` / `*.component.scss`. **Без inline `template`/`styles`.**
- **CSS-конвенція (обидва застосунки):** контекстний стиль — клас-корінь на секцію/компонент + стилі по тегах усередині; кілька спільних компонент-класів (`.btn`, `.card`, `.chip`) **без BEM-модифікаторів** (`--`/`__`); **без `#id`-селекторів**; низька, пласка специфічність (макс ~2-3 рівні вкладеності).
- **Форми (Angular):** Reactive Forms + `mat-form-field`.
- **i18n:** ключі в `uk`/`en`, без захардкоджених рядків.
- **Web:** статика-перш-за-все — контент пре-рендериться на білді, JS лише оживляє; розмітка не живе HTML-рядками в JS.
Пиши стисло й конкретно, щоб майбутні сесії Claude Code це читали й дотримувались.

## 1. Рефактор компонентів
- Пройтись по `apps/cabinet/src/**`: усі компоненти з inline-шаблонами/стилями розкласти на окремі `.html` та `.scss`.
- Перевести SCSS на контекстну конвенцію (як у п.0): прибрати BEM-нейминг, scoping через клас-корінь компонента, теги всередині. Зберегти візуал 1:1.
- Component styles лишаються інкапсульованими (ViewEncapsulation за замовчуванням) — це ок; конвенція стосується нейзингу всередині.

## 2. Angular Material + reactive forms
- Підключити Angular Material (тема під наявні кольори — кастомна палітра/CSS-перемінні проєкту, не дефолтний фіолетовий Material).
- Форми кабінету (насамперед **форма події** B3 і **профіль** B4) перевести на `mat-form-field` + reactive controls: інпути, textarea, **`mat-select`** для Категорії/Місто (нормально стилізований, замість нативного `<select>`), toggles для ціни.
- Зберегти наявну валідацію й повідомлення про помилки (через `mat-error`).

## 3. Баг: у полі «Місто» лише Львів
- Зʼясувати причину: чи `GET /cities` повертає лише Львів (тоді це дані сиду — за потреби додати міста через адмінку/сид), чи відмалювання прив'язане неправильно і не показує всі повернені міста.
- Якщо проблема в прив'язці — виправити (показувати всі міста з API). Якщо в даних — лишити коректний код, а нові міста додавати через адмінку (D2).

## 4. Динамічне прев'ю
- Прев'ю картки події має оновлюватись **наживо** при зміні форми (зараз статичне). Підписатись на `form.valueChanges` і ребіндити прев'ю (назва, категорія-бейдж, дата, ціна/Free, місто, cover).
- Дебаунс невеликий (напр. 150-200мс), щоб не смикалось на кожну літеру.

## 5. Мапа: автопідказки адреси + пошук місця
- У секції «Де» підключити **Google Places Autocomplete** на полі «Адреса»: при введенні вулиці показувати підказки; вибір підказки → проставляти `lat`/`lng`, центрувати маркер і мапу на це місце, заповнювати адресу.
- Зворотно: перетягування маркера → оновлювати `lat`/`lng` (і за можливості reverse-geocode в адресу).
- Потрібні API в Google: **Maps JavaScript API**, **Places API**, **Geocoding API** (увімкнути на ключі; обмежити по referrer — як домовлялись).

## 6. Зміна міста → рецентр мапи
- При зміні `mat-select` «Місто» мапа має центруватись на вибране місто (координати міста). Якщо в моделі `City` немає `lat/lng` — або додати їх (узгодити з API/сидом), або тимчасово мати словник координат міст на фронті; познач обраний підхід у коментарі.

## 7. Анімації
- Додати делікатні **fadeIn** при появі екранів/секцій (Angular animations): дашборд-плитки, список подій, відкриття форми, прев'ю. Тримати тонко (≤200-300мс, без надмірності).

## Обмеження / перевірка
- Не чіпати `apps/api` (хіба що п.3/п.6 вимагатимуть полів — тоді окремо познач, що потрібна зміна схеми, але **не** роби міграцію в цьому промпті без явної потреби; якщо потрібна — опиши її в підсумку).
- Зберегти cookie-auth, ролі, i18n, наявні маршрути й функціонал.
- `ng build` — чисто; `ng lint`/`tsc --noEmit` — без помилок.
- Ручний чекліст наприкінці: форма події (Material-поля, mat-select міста/категорії), живе прев'ю, автопідказки адреси, рецентр мапи при зміні міста, fadeIn, вигляд відповідає попередньому дизайну.
- Підсумок: список розкладених компонентів, що перейменовано в CSS, які Google API треба ввімкнути, чи потрібні зміни в API/схемі (п.6).

> Примітка: Places/Geocoding потребують відповідних API на Maps-ключі та білінгу в Google Cloud. Якщо ключ ще не має цих API — познач у підсумку, що їх треба ввімкнути.
