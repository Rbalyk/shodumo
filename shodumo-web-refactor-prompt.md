# ShoDumo — Рефактор публічного сайту (промпт для Claude Code)

> Запусти Claude Code в корені monorepo `shodumo`. Зачіпає **тільки `apps/web`**.
> Кабінет (`apps/cabinet`, Angular) і API (`apps/api`) **не чіпати** — кабінет за логіном, SEO там не діє.

Мета — зробити код зрозумілішим і дружнішим до SEO: менше розмітки в JS, більше готового HTML; стилі — контекстні замість BEM-модифікаторів. **Поведінка й вигляд сайту мають лишитись ті самі** — це рефактор, не редизайн.

---

## Частина 1. Рендер стрічки — гібрид (build-time HTML + клієнтська догідрація)

### Проблема зараз
`home.js` будує картки подій рядками HTML у JS і вставляє в DOM на завантаженні. Тобто початкова стрічка — порожня в HTML (погано для SEO й першого рендеру), а розмітка живе всередині JS.

### Що зробити
1. **Єдине джерело шаблону картки.** Винеси розмітку картки події в **один** спільний шаблон, який використовується і на білді, і на клієнті. Підхід на вибір (обери чистіший для цього стеку): build-партіал, який рендериться у прихований `<template id="event-card-tpl">` у HTML, плюс маленька JS-функція, що клонує шаблон і заповнює дані з `data-`-слотів. **Жодних довгих HTML-рядків у JS.**
2. **Пре-рендер стрічки на білді.** Розшир наявний `gulp prerender` (той, що вже генерує `event/:slug` та `organizer/:id`): нехай він фетчить опубліковані події (`GET /events`, обидві мови) і **вкладає готові картки стрічки прямо в `index.html`** (і в `dist/en/index.html`). Так контент одразу в HTML → SEO + швидкий перший рендер.
3. **Клієнтська догідрація замість повного перемалювання.** `home.js` на завантаженні **не стирає** пре-рендерену стрічку, а:
   - навішує інтерактив на наявний DOM (фільтр-чіпи, перемикач Список/Карта, кнопки «Піду»/«Зберегти»);
   - один раз тихо фетчить актуальні дані й **оновлює** стрічку (свіжість: нові події/лічильники), використовуючи той самий шаблон картки;
   - при зміні фільтра/міста — фетчить підмножину й перерендерює лише сітку.
4. **Свіжість статичного снапшоту (опційно, познач у README).** Клієнтська догідрація вже дає свіжі дані відвідувачу. Для оновлення *статичного* HTML між білдами — рекомендований шлях: **Cloudflare Pages Deploy Hook**, який смикається при публікації/модерації події. Реалізацію хука лиши як нотатку на потім, у код зараз не тягни.

### Не зламати
- Наявний пайплайн: `gulp build`, `gulp prerender`, i18n-токени `@@t.*`, інжект `API_URL/SITE_URL/APP_URL`, hreflang/canonical, sitemap.
- Пре-рендер сторінок події/організатора, JSON-LD, OG-теги.
- Після білда: у `dist/index.html` стрічка присутня як **статичний HTML**; у `dist/js/*.js` **немає** HTML-рядків карток і немає `localhost`.

## Частина 2. CSS — контекстний стиль + кілька компонент-класів

### Конвенція (прийнята)
- **Корінь-секція = клас**, всередині — стилі по **тегах** і вкладеності. Приклад замість BEM:
  ```scss
  // було: .feed__count { } .event-card__title { }
  .feed {
    .count { color: var(--ink-2); }
    article {
      h3 { font-size: 18px; }
      .price { font-weight: 700; }
    }
  }
  ```
- **Кілька спільних компонент-класів лишаємо** (`.btn`, `.card`, `.chip`) — щоб не дублювати. Але **без BEM-модифікаторів** (`--grad`/`--soft`/`__element`). Варіанти кнопок — окремими простими класами: `.btn` (база) + `.btn-accent`, `.btn-soft`, `.btn-outline` (а не `.btn--grad`).
- **Не використовувати `#id`-селектори** для стилів (висока специфічність). Scoping-корінь — завжди клас секції.
- Тримати специфічність низькою й пласкою; уникати глибокої вкладеності (макс ~2-3 рівні).

### Що зробити
1. Пройтись по `src/scss/**` і шаблонах `src/pages/**`: прибрати BEM `block__element--modifier`-нейминг, перевести на секція-корінь + теги.
2. Перейменувати модифікатори кнопок (`.btn--grad` → `.btn-accent` тощо) і оновити всі місця використання в HTML/JS-шаблонах.
3. Зберегти візуал 1:1 (ті самі кольори/відступи через наявні CSS-перемінні `--grad`, `--ink`, `--surface-*` тощо).
4. Семантичний HTML (`<article>`, `<nav>`, `<h1-3>`, `<time>`), `alt`, фокус-стани, aria для модалки — зберегти/покращити.

## Порядок і перевірка
1. CSS-рефактор (секція за секцією: topbar/header → feed → event-page → organizer → modal → footer), звіряючи візуал.
2. Винесення шаблону картки в єдине джерело + догідрація в `home.js`.
3. Розширення `gulp prerender` на інжект стрічки в `index.html` (uk + en).
4. `npm run build && npm run prerender` (з `API_URL=https://api.shodumo.com` тощо) — без помилок.
5. Чеклист:
   - `dist/index.html` містить статичні картки стрічки (не порожній контейнер);
   - `grep -r "localhost" dist/` — порожньо;
   - у `dist/js/*.js` немає HTML-рядків карток;
   - сайт виглядає й поводиться як раніше (фільтри, карта, «Піду», логін, перемикач мови, кнопка «Створити подію»);
   - 0 нерезолвлених `@@`-токенів; en-гілка без українських UI-рядків.

Наприкінці — короткий підсумок: що винесено з JS, які класи перейменовано, як тепер влаштований рендер стрічки.
